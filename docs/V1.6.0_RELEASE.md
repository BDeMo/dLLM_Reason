# dLLM-Reason v1.6.0 Release Notes

**Date**: 2026-04-20
**Branch**: `main` @ `d6cb384` (merged from `dev`)

---

## Headline

v1.6.0 introduces the **Canvas-Structured SFT (T6)** training pipeline: an
AR-teacher (Qwen3 / Qwen3.5 family) generates structured reasoning traces
(`<SETUP>` / `<STEP_x>` / `<ANSWER>`) on GSM8K train prompts, and LLaDA-8B
is fine-tuned on those traces via the standard masked-diffusion SFT loss
restricted to the answer region.

This release is the first training-side contribution that operationalises
the **canvas-constrained reasoning** framing established across v1.5.x
by the exhaustive A-axis / H3 / P6 exploration phase.

## What's new

### Training pipeline (`v1.6` core)

- `scripts/run_v1.6.sh` ŌĆö end-to-end orchestrator with four phases:
  - **0** Downloads (Qwen teacher + GSM8K train)
  - **1** T6 canvas distill (teacher trace + SFT)
  - **3** Eval on held-out `scope_fail` + `scope_ok`
  - **4** Archive stub to `docs/archive/finding_v1.6_*.zh.md`
  - Flags: `--t6_gpus N`, `--smoke`, `--mirror`,
    `--resume_t6_run_dir`, `--force_retrain_t6`, `--include_t7` (legacy),
    `--from_phase`, `--to_phase`, `--skip_qwen`, `--skip_gsm8k`,
    `--dry_run`, `--check_only`.

- `scripts/run_t6_shards.sh` ŌĆö multi-GPU teacher trace orchestrator
  (pattern mirrors the v1.5 `run_ss_shards.sh`).

- `scripts/setup_env_v1.6.sh` ŌĆö fresh-venv bootstrap with
  transformers-from-master (required for Qwen3.5 architecture).

- `scripts/download_qwen.py` ŌĆö Qwen3.5 / Qwen3 / Qwen2.5 candidates with
  HF mirror support (`--mirror hf-mirror | modelscope | <url>`) and
  post-download file verification.

- `scripts/validate/load_gsm8k_train.py` ŌĆö HF gsm8k train ŌåÆ scope JSON
  with mirror support and schema verification.

- `scripts/validate/t6_teacher_trace.py` ŌĆö local / OpenAI / Anthropic
  teacher with strict-format parsing, retry, per-prompt atomic writes
  and `--prompt_start/--prompt_end/--skip_aggregate` for sharding.

- `scripts/validate/t6t7_train.py` ŌĆö Finetuner wrapper with HF-format
  checkpoint export alongside `.pt` (so `serve.py` and `v16_eval.py`
  can load trained weights via `AutoModel.from_pretrained`).

- `scripts/validate/v16_eval.py` ŌĆö T=0 greedy pass@1 eval on held-out
  `scope_fail` + `scope_ok`, with FAIL18 and ceiling-5 breakdown +
  comparison markdown table across ckpts.

- `src/dllm_reason/data/jsonl_dataset.py` ŌĆö JSONL ŌåÆ Dataset loader
  aligned with the existing Finetuner pipeline (stratified
  train/val split, in-memory records).

### Infrastructure

- `src/dllm_reason/utils/llada_checkpoint_patch.py` (already in v1.5.x
  but now stable) ŌĆö idempotent, self-healing compat patch for LLaDA

- `src/dllm_reason/models/llada.py` ŌĆö shim for `torch_dtype` / `dtype`

### Deprecations / scope notes

- **T7 self-distill** (sample + filter correct ŌåÆ SFT) is retained in
  the codebase (`scripts/validate/t7_gen_correct_samples.py`) but not
  part of the v1.6 default pipeline. Use `--include_t7` to opt in;
  production validation will be in a later release.

- **Strategy Search (B1 / SS)** from v1.5.x is preserved
  (`scripts/validate/strategy_search.py`, `run_ss_shards.sh`,
  `ss_analyze.py`, `ss_to_sft.py`, `ss_oracle_prior.zh.md`) but
  superseded as the active phase-3 focus. See
  `docs/plans/2026-04-19_ss_run_postmortem.zh.md` for the wrap.

## Quick start

```bash
# Fresh venv with Qwen3.5-compatible transformers
bash scripts/setup_env_v1.6.sh --mirror
source .venv_v16/bin/activate

# Or without a fresh venv, use Qwen3-8B teacher (transformers < 5 OK)
python scripts/download_qwen.py --mirror hf-mirror --sizes 3-8B

# End-to-end v1.6 run on 8 GPUs
bash scripts/run_v1.6.sh --mirror hf-mirror --t6_gpus 8 \
    --teacher_size 3-8B              # or 3.5-4B / 3.5-9B / 3.5-27B

# Smoke (200 prompts, ~20 min)
bash scripts/run_v1.6.sh --smoke --mirror hf-mirror --t6_gpus 8

# Resume after interruption
bash scripts/run_v1.6.sh --t6_gpus 8 \
    --resume_t6_run_dir runs/validation/t6_teacher_trace_<ts>
```

## Dependencies

- Python Ōēź 3.10
- torch with CUDA
- transformers Ōēź 4.56 for Qwen3; transformers 5.x (master) for Qwen3.5
- accelerate, datasets, huggingface_hub, safetensors
- fastapi, uvicorn (for serve.py)

## Known limitations

- T6 SFT is single-GPU in this release. Multi-GPU DDP wrapping is the
  primary perf target for v1.6.1.
- Phase 4 archive stub is a template; numbers table is auto-embedded
  from `comparison.md` but the narrative ("ń╗ōĶ«║") section is filled
  manually after reviewing eval output.
- T6 teacher currently uses HF `transformers.pipeline` serial
  generation. vllm backend is not wired; see
  `docs/plans/2026-04-19_v1.6_plan.zh.md` for the roadmap note.
