## 目标

v1.6 release 的核心训练工作：把 "探索到的知识"（A 轴 SUPPORTED、H3 pass@8 高、ceiling 5 是 multi-step tracking 错）翻译成**两条 SFT 流水线**：

- **T7 Self-Distill**：LLaDA 自己 pass@8 sampling 过滤 correct → SFT
- **T6 AR-Teacher Canvas Distill**：GPT-4/Qwen 生成结构化 trace → SFT with canvas-friendly structure

两条组合成 **2-stage distill**：T7 做 stage-1 push pass@1，T6 做 stage-2 破 ceiling。

## Quantitative target

| 指标 | Baseline | v1.6 Target | 根据 |
|---|---|---|---|
| Pass@1 at T=0 (fail set) | 0% | **60%+** | H3 pass@1 at T=1.0 = 71.7%，SFT 应能 recover 大部分 |
| Pass@1 overall (n=60 fail+49 ok) | ~89.6% | **92%+** | 小幅全局提升 |
| FAIL18 rescue | 13/18 | **14-16/18** | 结构化 trace 救 1-3 条 ceiling |
| Ceiling 5 break | 0/5 | **1-2/5** | AR teacher 能对这些题 |

全部 **training-free at inference**（SFT 完后正常 T=0 greedy 推理，不用多 sampling / LoRA 切换）。

## Execution 三阶段

### Phase 1 —— T7 Self-Distill（1-2 天）

**Step 1.1** 生成 T7 训练数据（需要 GPU，启一个 serve.py）：
```bash
# 先起 serve
CUDA_VISIBLE_DEVICES=0 python scripts/serve.py --port 8000 &

# 跑 sampling + filter
python scripts/validate/t7_gen_correct_samples.py \
    --groups fail,ok --n 60 \
    --temperatures 0.3,0.7,1.0 --n_samples 8 \
    --gen_length 192 --block_length 32 \
    --pick shortest \
    --server_url http://127.0.0.1:8000
```

**预期耗时**：109 prompts × 24 samples × 5s ≈ 3.5h

**预期 cover rate**：按 H3 pass@8=86.67%，约 95/109 prompts 有 ≥1 correct sample

**Step 1.2** T7 SFT training：
```bash
python scripts/validate/t6t7_train.py \
    --jsonl_path runs/validation/t7_selfdistill_<ts>/t7_sft.jsonl \
    --run_name t7_stage1 \
    --max_steps 2000 --batch_size 4 --lr 2e-5
```

**预期耗时**：~1h on A100（2000 steps × 16 effective batch × 512 seq）

**Step 1.3** 评估：用新 ckpt 在 fail/ok 集跑 T=0 pass@1

### Phase 2 —— T6 AR-Teacher Canvas Distill（2-3 天）

**Step 2.1** Teacher trace 生成（OpenAI API 或本地 Qwen-32B）：
```bash
# 小规模先测 FAIL18 + ceiling5
python scripts/validate/t6_teacher_trace.py \
    --prompt_indices fail18 \
    --teacher openai --openai_model gpt-4o-mini \
    --retries_per_prompt 3

# 大规模：所有 fail + ok
python scripts/validate/t6_teacher_trace.py \
    --groups fail,ok --n 60 \
    --teacher openai --openai_model gpt-4o
```

**预期耗时**：109 prompts × 3 retries × ~2s = 10 min（API 很快）
**预期 accept rate**：~85-95%（GPT-4o 在 gsm8k 大部分能做对并按 format 输出）

**Step 2.2** T6 SFT，warm-start from T7 ckpt：
```bash
python scripts/validate/t6t7_train.py \
    --jsonl_path runs/validation/t6_teacher_trace_<ts>/t6_sft.jsonl \
    --run_name t6_stage2 \
    --init_ckpt runs/training/t7_stage1/best.pt \
    --max_steps 1500 --batch_size 4 --lr 1e-5
```

**预期耗时**：~45 min on A100

**Step 2.3** 评估 + 对比（T7-only vs T7→T6）

## 风险 & decision points

| 风险 | 触发条件 | 降级方案 |
|---|---|---|
| T7 SFT overfit 到 sampled correct 的短 output | val loss 不降 / pass@1 不升 | 换 `--pick longest` 或 mix policy |
| Teacher trace 格式遵守率低 | T6 accept rate < 50% | 换 GPT-4o（更 instruct-following），加 retry |
| LLaDA FT 不收敛 | loss 发散 | 降 lr 到 1e-5；或锁 embedding，只训 MLP |
| Pass@1 涨，但 canvas-unique rescue 没涨 | FAIL18 rescue 不变 | v1.6 story 简化为"self-distill + teacher"，暂不强调 canvas；future work |
