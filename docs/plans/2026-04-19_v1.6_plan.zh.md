## 目标

v1.6 release 的核心训练工作：把 "探索到的知识"（A 轴 SUPPORTED、H3 pass@8 高、ceiling 5 是 multi-step tracking 错）翻译成**两条 SFT 流水线**：

- **T7 Self-Distill**：LLaDA 自己 pass@8 sampling 过滤 correct → SFT
- **T6 AR-Teacher Canvas Distill**：GPT-4/Qwen 生成结构化 trace → SFT with canvas-friendly structure

两条组合成 **2-stage distill**：T7 做 stage-1 push pass@1，T6 做 stage-2 破 ceiling。

## Training Objectives（两条 pipeline 共享 loss，语义不同）

### 通用 SFT loss （LLaDA masked diffusion 范式）

LLaDA 训 loss 不是 AR next-token prediction，是 **masked LM NLL**。给定样本 `x_0 = concat(prompt_tokens, answer_tokens)`：

```
1. Sample timestep t ∈ [0, 1]
2. x_t = noise(x_0, t)  — 按 t 比例随机 mask tokens
3. logits = model.forward(x_t, t)
4. L = -E_{i ∈ (masked positions) ∩ (answer positions)}
            [log p_θ(x_0^i | x_t, t)]
5. backward + step
```

关键约定（Finetuner 已实现 `loss_on_answer_only=True`）：
- **只对 answer 区算 loss**（prompt 位置永远不被 mask、不进 loss）
- Timestep t 控制 mask 比例（t≈0 → 小部分 mask，t≈1 → 几乎全 mask）
- 每个 step 模型练习在**不同 mask 密度**下的 answer 填充

### T7 训练目标

**Data**: `(prompt, self-sampled correct output)` pair
- Prompt = 原始 gsm8k train 题
- Answer = LLaDA 自己在 T ∈ {0.3, 0.7, 1.0}, N=8 下采到的 **答对** sample（per-prompt 选一条）

**Semantic objective**: Concentrate probability mass onto correct answers.

证据链条：
- H3 实测 pass@1 at T=1.0 = 71.7% → **LLaDA 内部已对正确答案分配非零概率**
- Pass@1 at T=0 (greedy) = 0% → 正确答案不是 argmax，但在概率分布 top-K 内
- SFT 用正确 sample 为 target → 参数更新 → argmax 变成正确答案
- 本质：把"偶尔采对" 变"稳定 greedy 对"

### T6 训练目标

**Data**: `(prompt, teacher structured trace)` pair
- Prompt = 原始 gsm8k train 题
- Answer = AR teacher (GPT-4 / Qwen-32B) 生成的 `<SETUP>...</SETUP><STEP_1>...</STEP_1>...<ANSWER>...</ANSWER>` 结构化解法，且 `<ANSWER>` 匹配 gt

**Semantic objective**: Inject external AR reasoning via canvas-friendly structure.

LLaDA 学到三件事：
1. 结构 markers 的**canvas 位置** (`<SETUP>` 靠前，`<ANSWER>` 靠末尾)
2. 每个 section 内部**该写什么内容** (e.g. `<STEP_1>` 写第一步计算)
3. 跨 section 的**推理衔接** (`<STEP_N>` 依赖 `<STEP_{N-1}>` 结果)

**为啥不只靠 T7**：T7 正样本继承 LLaDA 自己的 reasoning 弱点（ceiling 5 这种 multi-step tracking 错）。T6 引入**外部 teacher 的 reasoning 能力**，通过 canvas structure 这种 diffusion LM 友好的形式注入。两者叠加：T7 调整概率 mass，T6 注入结构化 reasoning pattern。

## Algorithms（pseudocode）

### Stage 0 — Data generation

**T7 self-sampled correct**:
```python
for prompt, gt in gsm8k_train:
    correct = []
    for T in [0.3, 0.7, 1.0]:
        for _ in range(N_samples=8):
            out = LLaDA.sample(prompt, T=T, gen_length=192, block=32)
            if is_correct(out, gt):
                correct.append({'output': out, 'T': T})
    if correct:
        best = pick_shortest(correct)
        emit_jsonl({'question': prompt, 'answer': best['output']})
```

**T6 teacher structured trace**:
```python
for prompt, gt in gsm8k_train:
    for attempt in range(retries=3):
        trace = teacher_LM.generate(
            system=TEACHER_SYSTEM,
            user=TEACHER_USER_TEMPLATE.format(prompt=prompt),
            temperature=0.0,
        )
        ans = extract_teacher_answer(trace)
        sections = parse_sections(trace)
        if is_correct(ans, gt) and 'SETUP' in sections and 'ANSWER' in sections:
            emit_jsonl({'question': prompt, 'answer': trace, 'sections': sections})
            break
```

### Stage 1 — T7 SFT

```python
model = LLaDA.from_pretrained('GSAI-ML/LLaDA-8B-Instruct')
ds = JsonlReasoningDataset(t7_jsonl, tokenizer, max_seq_len=512)
loader = DataLoader(ds, batch_size=4, shuffle=True)
opt = AdamW(model.parameters(), lr=2e-5)
scheduler = CosineAnnealingWarmRestarts(opt, T_0=1000)

for step in range(max_steps=3000):
    batch = next(loader)
    x_0 = batch['input_ids']
    prompt_mask = batch['prompt_mask']
    
    t = torch.rand(B).clamp(1e-5, 1-1e-5)
    x_t = LLaDA.noise_input(x_0, t)
    logits = model.forward(x_t, t)
    log_probs = F.log_softmax(logits, dim=-1)
    nll = -log_probs.gather(-1, x_0.unsqueeze(-1)).squeeze(-1)
    
    loss_mask = (x_t == MASK_ID) & ~prompt_mask
    loss = (nll * loss_mask.float()).sum() / loss_mask.float().sum().clamp(min=1)
    loss.backward(); clip_grad(1.0); opt.step(); scheduler.step(); opt.zero_grad()
    
save ckpt → runs/training/t7_stage1/best.pt
```

### Stage 2 — T6 SFT (warm-start from Stage 1)

同 Stage 1 的 loop，但：
- `init_ckpt = runs/training/t7_stage1/best.pt`
- `lr = 1e-5` （warm-start 用更小 lr）
- `max_steps = 2000`
- Data = T6 JSONL

→ `runs/training/t6_stage2/best.pt` = **final v1.6 ckpt**

### Evaluation

```python
for ckpt in [baseline_LLaDA, ckpt_t7, ckpt_final]:
    for prompt in test_set:   # 60 fail + 49 ok, held out
        output = ckpt.sample(prompt, T=0, gen=192, bl=32)
        correct = is_correct(output, gt)
    report:
      - Pass@1 T=0 on fail subset
      - Pass@1 T=0 on ok subset (regression guard)
      - FAIL18 rescue count
      - Ceiling 5 break count
```

**Test set 绝不进训练**（train/test split 天然分离：我们 test 是 gsm8k test，training 是 gsm8k train）。

## Data Strategy —— Fast vs Full

### Fast 档（先跑，验证方法可行）

**用现有** `runs/validation/scope_fail_prompts.json` + `scope_ok_prompts.json` = **109 prompts**

- **作用**：验证 pipeline 跑通；快速看 training loss 是否正常下降、pass@1 是否提升
- **问题**：数据量太小（109 × 0.9 accept ≈ 100 SFT pairs），过拟合风险高
- **缺陷**：跟 test set 完全重叠（因为 scope 本身就是 test set 的 fail/ok 分类）→ **不能拿来报 eval 数字**（相当于 train on test）
- **只用于快速验证代码是否 work**，不作为 v1.6 release 训练

### Full 档（正式）—— **推荐用这个作为 v1.6 正式训练**

**用 GSM8K train 原始集** = **7473 prompts**

| 项 | 数字 |
|---|---|
| 原始数据集 | HuggingFace `openai/gsm8k`, split=`train` |
| 规模 | 7473 problems |
| Test overlap | **零**（gsm8k train/test 官方分离）|
| T7 cover 预估 | ~6500/7473 prompts 有 ≥1 correct sample （baseline + H3 pass@8）|
| T6 accept 预估 | ~7000/7473 traces （GPT-4o 解 gsm8k train 非常强）|

### Test set（永不训）

- 现有 60 fail + 49 ok （`runs/validation/scope_*.json`）
- 所有 v1.6 vs baseline 的对比数字都在这上面算
- 保证跟 A 轴 / H3 / SS 的历史数字同口径可比

### Phase 1 —— T7 Self-Distill（1-2 天）

**Step 1.1** 生成 T7 训练数据（需要 GPU，启一个 serve.py）：
```bash
# 先起 serve
CUDA_VISIBLE_DEVICES=0 python scripts/serve.py --port 8000 &

# 跑 sampling + filter
python scripts/validate/t7_gen_correct_samples.py \
    --groups fail,ok --n 60 \
    --temperatures 0.3,0.7,1.0 --n_samples 8 \
    --gen_length 192 --block_length 32 \
    --pick shortest \
    --server_url http://127.0.0.1:8000
```

```bash
python scripts/validate/t6t7_train.py \
```

**预期耗时**：~1h on A100（2000 steps × 16 effective batch × 512 seq）

## 风险 & decision points

| 风险 | 触发条件 | 降级方案 |
|---|---|---|
| T7 SFT overfit 到 sampled correct 的短 output | val loss 不降 / pass@1 不升 | 换 `--pick longest` 或 mix policy |
| Teacher trace 格式遵守率低 | T6 accept rate < 50% | 换 GPT-4o（更 instruct-following），加 retry |
| LLaDA FT 不收敛 | loss 发散 | 降 lr 到 1e-5；或锁 embedding，只训 MLP |
| Pass@1 涨，但 canvas-unique rescue 没涨 | FAIL18 rescue 不变 | v1.6 story 简化为"self-distill + teacher"，暂不强调 canvas；future work |
