# dLLM-Reason v1.6.1 Release Notes

**Date**: 2026-04-20
**Branch**: `main` (merged from `dev`)

---

## Headline

v1.6.1 is a **correctness + scale** release for the v1.6 canvas-distill
pipeline. Every major issue that surfaced in the v1.6.0 test runs is
addressed, three capability upgrades land, and one master script
(`scripts/run_all_v1.6.1.sh`) now does the whole pipeline end-to-end.

## What's fixed

### 🔴 Eval config drift (the big one)

**v1.6.0 bug**: `v16_eval.py` defaulted to `gen_length=192`, but
`scope_fail_prompts.json` was generated with the canonical
`gen_length=128`. At g=192 the baseline trivially "solves" ~65% of the
supposed-to-be-hard fail prompts purely from longer canvas (A6 effect),
producing a meaningless comparison table where both baseline and T6
scored 65% identically.

**Fix**:
- `v16_eval.py` defaults aligned to canonical: `gen_length=128`,
  `block_length=32`, `temperature=0`, `remasking="low_confidence"`.
- Sanity check (`model.generate("What is 2+2?")`) before the main loop.
  If broken, FATAL-raises immediately instead of producing 109 empty
  outputs and a bogus 0% comparison.
- First-prompt exception now re-raises instead of silently setting
  `out=""`.
- `max_seq_len` scaled to `max(512, gen_length + 256)` so prompt + gen
  always fit.

See `docs/archive/issues/2026-04-20_eval_config_drift.zh.md`.

### 🔴 Scope files may have drifted from current model

The scope files in `runs/validation/scope_*.json` came from an old
research DB and metadata indicates `num_steps=128 / block_length=32`,
but the "correct" partition may no longer match current LLaDA-Instruct.

**Fix**: New `scripts/validate/regen_scope.py` regenerates both
`scope_fail_prompts.json` and `scope_ok_prompts.json` from the gsm8k
test split (1319 prompts) using the canonical config against the
current `GSAI-ML/LLaDA-8B-Instruct`. Fully idempotent + resumable +
supports `--prompt_start/--prompt_end` sharding.

### 🟡 Teacher trace chat-template noise (now also at save time)

v1.6.0 introduced `clean_trace()` and applied it in `t6_teacher_trace.py`
at save time so `per_prompt/*.json` and `t6_sft.jsonl` are already
clean. v1.6.1 retains this + the post-hoc `t6_clean_jsonl.py` as backup.

### 🟡 Miscellaneous minor bugs

See the running log at
`docs/archive/issues/minors.zh.md`:
- `_llada` attribute name for HF-export
- `requires_grad=False` default after `from_pretrained`
- `build_jsonl_dataset` single-record edge case
- `$GROUPS` bash readonly collision
- stdout buffering → empty shard logs

## What's new

### ⭐ Multi-output teacher mode (`--collect_all_valid` default True)

Previously, each prompt produced **one** accepted teacher trace (the
first valid retry). In v1.6.1 the default behaviour is to try ALL
`--retries_per_prompt` rounds at `--temperature > 0` and keep EVERY
valid trace per prompt. Each valid trace becomes one SFT row.

Given Qwen3-8B teacher at T=0.7 with 5 retries:
- Previously: ~1600 SFT rows for 2000 gsm8k train prompts
- v1.6.1: ~4000-6000 diverse SFT rows (same prompt, multiple valid
  reasoning paths)

De-duplicated at the trace-text level (identical traces collapsed).

Applies identically to **any dataset** — the mechanism is not gsm8k-
specific. Point `--scope_path` at another dataset's prompts JSON and
the same multi-output collection works.

Pass `--first_valid_only` to restore the pre-v1.6.1 one-trace-per-prompt
behaviour.

### ⭐ Canonical unmasking-order documentation

New `docs/archive/unmasking_order_v1.6.1.zh.md` makes explicit:

- Training: unmask order is **implicit random** (MDLM standard)
- Inference: **low_confidence greedy** (LLaDA default `remasking`)
- Canvas structure (`<SETUP>/<STEP_N>/<ANSWER>`) **indirectly**
  influences inference order via SFT-learned confidence distribution
  — we do NOT explicitly modify the scheduler
- Contrast table vs LogicDiff / DAWN / Where-to-Unmask

### ⭐ `scripts/run_all_v1.6.1.sh` — single-command big run

End-to-end six-phase pipeline:
0. Download Qwen + gsm8k train
1. Start LLaDA serve
2. Regen scope (canonical config)
3. T6 teacher trace (8-GPU, multi-output)
4. T6 SFT (single GPU, auto-skip if ckpt exists)
5. Eval baseline vs T6 (canonical config)
6. Archive stub

Every phase idempotent. Re-run after interrupt picks up where it
stopped. Designed for overnight / multi-hour 8×A100 runs.

```bash
bash scripts/run_all_v1.6.1.sh
# or with China mirror
bash scripts/run_all_v1.6.1.sh --mirror hf-mirror
# or skip downloads (already have everything)
bash scripts/run_all_v1.6.1.sh --from_phase 2
```

## Defaults changed

| Setting | v1.6.0 | v1.6.1 | Reason |
|---|---|---|---|
| `v16_eval.py` `gen_length` | 192 | **128** | Canonical alignment |
| `t6_teacher_trace` `temperature` | 0.0 | **0.7** (via `run_all`) | Enable output diversity |
| `t6_teacher_trace` `retries` | 3 | **5** (via `run_all`) | More diverse traces |
| `t6_teacher_trace` valid traces per prompt | 1 | **all valid** | Multi-output mode |
| `t6t7_train` `max_seq_len` | 512 | **768** | Fit prompt+target safely |

## Issues archive

All issues encountered during v1.6.x development are now tracked under:

- `docs/archive/issues/minors.zh.md` — running log of small bugs,
  append-only, one line per fix
- `docs/archive/issues/YYYY-MM-DD_<name>.zh.md` — major issues (one
  file each, with symptom / root cause / fix / lessons)

## Dependencies

Unchanged from v1.6.0. For Qwen3.5 teacher, transformers 5.x
(from-source) required; for Qwen3 / Qwen2.5 teacher, stable
transformers 4.56–5.0 suffices. See `scripts/setup_env_v1.6.sh`.

## Known limitations

- T6 SFT still single-GPU. Multi-GPU DDP/FSDP is the main v1.6.2
  target.
- Phase 6 archive stub fills numbers from `comparison.md` but leaves
  the narrative 结论 section as TODO (manual review after each run).
- Eval doesn't yet auto-read canonical config from scope file
  metadata (it respects the CLI default, which is now correct; but a
  mismatch between user-supplied flags and scope file won't error —
  it should, in v1.6.2).

## Upgrade path from v1.6.0

```bash
git pull origin main
bash scripts/run_all_v1.6.1.sh --from_phase 2   # regen scope + everything downstream
```

The old v1.6.0 artifacts (`scope_fail_prompts.json`, any
`v16_eval_*` directories with 65% numbers, `runs/training/v16_t6*`
ckpts trained on dirty data) should be regarded as **invalidated**.
v1.6.1 produces fresh artifacts under new names
(`v161_eval_*`, `v161_t6`) so the old ones don't collide.
